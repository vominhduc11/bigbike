# API_FLOW_MAP.md — bigbike.vn

End-to-end map of every required user flow: URL → template → JS → transport → handler → APIs → data → permission → response.

Notation:
- **T** (status): Implemented / Plugin-provided / Partial / Needs verification.
- All AJAX handlers POST to `/wp-admin/admin-ajax.php` with JSON response (`wp_send_json_success|error`).
- All AJAX handlers are registered with **both** `wp_ajax_<name>` AND `wp_ajax_nopriv_<name>` — i.e. exposed to guests.
- **None** of the custom AJAX handlers call `check_ajax_referer`, `wp_verify_nonce`, or `current_user_can`. Evidence: entire [inc/ajax-functions.php](files/wp-content/themes/bigbike/inc/ajax-functions.php).

## Cross-cutting notes

1. **No CSRF / nonce on any custom AJAX endpoint.** See F07, F08, F10, F11, F12, F15, F16, F17.
2. **Mini-cart rendering relies on a template part named `header-cart`** referenced by `get_custom_template('header-cart')` at [ajax-functions.php:306](files/wp-content/themes/bigbike/inc/ajax-functions.php#L306) — the file was not located in the snapshot at `template-parts/header-cart.php`. **Needs verification** that this template exists (otherwise remove-cart response returns empty HTML).
3. **Checkout JS dependency risk:** WooCommerce default JS (`woocommerce`, `wc-cart-fragments`, `wc-add-to-cart`) is globally deregistered in `bigbike_disable_woocommerce_loading_css_js`. The theme supplies replacements for add-to-cart and mini-cart but still relies on WC core JS for checkout field show/hide, coupon apply, address lookup, etc. **Needs verification** that checkout still fully works after that wholesale deregistration.
4. **GTM events** emitted inline from templates:
   - `view_item` in [content-single-product.php:38-52](files/wp-content/themes/bigbike/woocommerce/content-single-product.php#L38-L52)
   - `view_cart` in [cart.php:33-45](files/wp-content/themes/bigbike/woocommerce/cart/cart.php#L33-L45)
   - `purchase` in [thankyou.php:41-51](files/wp-content/themes/bigbike/woocommerce/checkout/thankyou.php#L41-L51)
5. **Polylang-aware post IDs** hard-coded:
   - login=7970, register=7968, forgot=10155 (resolved via `pll_get_post(...)` to the current language's translation).
6. **SEO title filter order:** `wpseo_title` filter (Yoast) is used even though RankMath is the active SEO plugin. Migration must re-test.
